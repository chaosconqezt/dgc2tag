# Tracks Panel Redesign

## Goal
Rewrite the tracks panel (TrackMatcher + MatchRow + SingleArtistTracks + MultiArtistTracks) into a clean, modern table layout with perfect column alignment and grouped toolbar.

## Current Problems
1. Toolbar: all controls crammed in one line with `·` separators — cluttered
2. No table structure — columns misalign between rows, especially artist row
3. Manual spacer hacks in MultiArtistTracks
4. Inline styles everywhere, no visual hierarchy
5. No row separation or hover feedback

## Design

### CSS Grid Table (MatchRow)
Replace flex with CSS Grid. Fixed columns ensure perfect alignment across all rows:

```
[chk 24px] [num 32px] [local name 1fr] [dur 48px] [sim 48px] [dur 48px] [num 32px] [remote name 1fr]
```

Grid defined once in CSS as `.track-row`. Both MatchRow and artist row use the same grid — columns stay aligned by definition.

### Toolbar (TrackMatcher)
Group controls into 3 sections separated by subtle borders:

```
[✓ write titles · ✓ filenames] │ [12/12 · 10 exact · 2 close] │ [ID3|filename · ✓ multi-artist · ✓ strip parens]
```

Each group is a flex container. Middle section auto-arranges. Right section wraps if needed.

### Row Styling
- `.track-row`: grid, padding 4px 8px, border-bottom 1px solid var(--border-light)
- `.track-row:hover`: background var(--input-bg)
- `.track-row.disabled`: opacity 0.5
- Artist sub-row: same grid, top border none (visually merged), slightly dimmer text

### MatchRow Changes
- Remove all inline style objects for layout
- Use `.track-row` class for grid
- Column classes: `.track-chk`, `.track-num`, `.track-name`, `.track-dur`, `.track-sim`
- Number column: monospace, small font
- Sim column: colored badge-like display

### MultiArtistTracks Changes
- Remove manual spacer spans (36px, 40px, 36px)
- Artist row uses same `.track-row` grid class
- Local artist in col 3 (same as local name), remote artist in col 8 (same as remote name)
- Artist text: dimmer color, slightly smaller
