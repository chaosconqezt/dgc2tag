# План: Плагинная архитектура для источников данных

## Цель
Сделать добавление нового источника (Spotify, Discogs, Bandcamp, Metal Archives) тривиальным: 1 новый файл + регистрация.

## Текущая проблема
Добавление MusicBrainz потребовало изменения 10 файлов. Каждый новый источник = ~200 строк бойлерплейта в 7 файлах.

## Решение: 6 шагов

---

### Шаг 1: Унифицировать SearchResult (общий интерфейс)

**Файл: `client/src/types.ts`**

`SearchResult` уже является де-факто общим типом (все `handleSelect*` производят его). Официализируем:

```ts
interface SearchResult {
  source: 'dgc' | 'deezer' | 'musicbrainz' | string;
  id: string;                    // нормализованный string ID
  albumName: string | null;
  artist: string;
  albumArtist: string;
  coverUrl: string | null;
  country: string | null;
  year: string | null;
  label: string | null;
  genres: string[];
  releaseType: string | null;
  trackCount?: number;
  tracks?: { num: string; artist: string; name: string; duration?: number }[];
  url: string;
  extraTags?: Record<string, string>;
  compilation?: boolean;
  // DGC-specific (оставляем для обратной совместимости)
  postId?: number;
  genreIds?: number[];
  tracklist?: string;
  notes?: string;
  // MusicBrainz-specific
  musicbrainzReleaseId?: string;
  // ...
}
```

**Сервер `server/src/scraper.ts`** — `mapPost()` добавляет `source: 'dgc'`, `id: String(postId)`
**Сервер `server/src/deezer.ts`** — `searchDeezer()` возвращает результаты с `source: 'deezer'`, `id: String(albumId)`
**Сервер `server/src/musicbrainz.ts`** — `searchMusicBrainz()` возвращает `source: 'musicbrainz'`, `id: releaseId`

Убрать `DeezerSearchResult` и `MusicBrainzSearchResult` как отдельные интерфейсы — всё через `SearchResult`.

### Шаг 2: SearchSource плагин-интерфейс

**Новый файл: `server/src/sources/types.ts`**

```ts
interface SearchSource {
  id: string;                    // 'dgc' | 'deezer' | 'musicbrainz' | ...
  label: string;                 // 'DGC' | 'Deezer' | 'MusicBrainz'
  accentColor: string;           // '#ef4444' | '#4ade80' | '#f97316'

  search(artist?: string, album?: string, query?: string): Promise<SearchResult[]>;
  getDetails?(id: string): Promise<SearchResult | null>;
}
```

**Новый файл: `server/src/sources/dgc.ts`** — обёртка над scraper.ts
**Новый файл: `server/src/sources/deezer.ts`** — обёртка над deezer.ts
**Новый файл: `server/src/sources/musicbrainz.ts`** — обёртка над musicbrainz.ts

Каждый источник = 1 файл, реализующий `SearchSource`.

### Шаг 3: Реестр источников + автогенерация routes

**Новый файл: `server/src/sources/index.ts`**

```ts
import { dgcSource } from './dgc.js';
import { deezerSource } from './deezer.js';
import { musicbrainzSource } from './musicbrainz.js';

export const sources: SearchSource[] = [dgcSource, deezerSource, musicbrainzSource];
```

**Изменение: `server/src/index.ts`**

Вместо ручных route для каждого источника — цикл:

```ts
for (const src of sources) {
  app.post(`/api/search-${src.id}`, async (req, res) => {
    const { artist, album } = req.body;
    const results = await src.search(artist, album);
    res.json(results);
  });
  if (src.getDetails) {
    app.get(`/api/${src.id}/:id`, async (req, res) => {
      const result = await src.getDetails(req.params.id);
      res.json(result);
    });
  }
}
```

Добавить новый источник = добавить 1 файл + 1 строку в `sources/index.ts`.

### Шаг 4: Клиент — generic source state

**Изменение: `client/src/hooks/useAppContext.tsx`**

Заменить per-source state на generic map:

```ts
// Было:
searchResults: SearchResult[];
deezerResults: DeezerSearchResult[];
mbrainzResults: MusicBrainzSearchResult[];
selectedResult: SearchResult | null;
selectedDeezer: DeezerSearchResult | null;
selectedMbrainz: MusicBrainzSearchResult | null;
dgcLoading: boolean;
deezerLoading: boolean;
mbrainzLoading: boolean;

// Стало:
sourceResults: Record<string, SearchResult[]>;    // { dgc: [...], deezer: [...], musicbrainz: [...] }
sourceLoading: Record<string, boolean>;           // { dgc: false, deezer: true, musicbrainz: false }
selectedResult: SearchResult | null;              // единый selected (уже нормализован)
```

**Действия:**
- `SET_SOURCE_RESULTS` — `{ source: string, results: SearchResult[] }`
- `SET_SOURCE_LOADING` — `{ source: string, loading: boolean }`
- `SET_SELECTED_RESULT` — `{ result: SearchResult | null }` (один на все источники)

**handleSearch** — вызывает `search()` для каждого источника параллельно:
```ts
for (const src of sources) {
  api.sourceSearch(src.id, artist, album)
    .then(data => dispatch({ type: 'SET_SOURCE_RESULTS', payload: { source: src.id, results: data } }))
    .finally(() => dispatch({ type: 'SET_SOURCE_LOADING', payload: { source: src.id, loading: false } }));
}
```

**handleSelectResult** — один универсальный обработчик (не три отдельных):
```ts
const handleSelectResult = (result: SearchResult) => {
  clearSelectionState();
  dispatch({ type: 'SET_SELECTED_RESULT', payload: result });
  // Если есть getDetails — подтянуть треки
  if (result.source !== 'dgc') {
    api.sourceGetDetails(result.source, result.id).then(full => {
      dispatch({ type: 'SET_SELECTED_RESULT', payload: { ...result, tracks: full.tracks, extraTags: full.extraTags } });
    });
  } else {
    loadAlbumDetails(Number(result.id));
  }
};
```

### Шаг 5: Клиент — generic UI компоненты

**Изменение: `client/src/components/SearchResults.tsx`**

Вместо хардкода трёх панелей — итерация по `sourceResults`:

```tsx
const sourceConfigs = [
  { id: 'dgc', label: 'DGC', color: '#ef4444' },
  { id: 'deezer', label: 'DEEZER', color: '#4ade80' },
  { id: 'musicbrainz', label: 'MBRAINZ', color: '#f97316' },
];

// В JSX:
{sourceConfigs.map(cfg => (
  sourceResults[cfg.id]?.length > 0 && (
    <SourcePanel key={cfg.id} label={cfg.label} color={cfg.color}
      results={sourceResults[cfg.id]} selectedId={selectedResult?.id}
      onSelect={handleSelectResult} />
  )
))}
```

**Удалить:** `DgcResults.tsx`, `DeezerResults.tsx`, `MusicBrainzResults.tsx`
**Замена:** один `SourcePanel.tsx` (параметризуется label + color + results)

### Шаг 6: writeUserDefinedText data-driven

**Изменение: `server/src/tagWriter.ts`**

```ts
// Было: 12 позиционных параметров
function writeUserDefinedText(current, country, releaseType, postId, deezerId, mbReleaseId, ...)

// Стало: 1 запись
function writeUserDefinedText(current: TxxxFrame[], fields: Record<string, string | undefined>): TxxxFrame[]
```

Каждый источник声明ит `tagMappings`:
```ts
// sources/dgc.ts
tagMappings: [
  { description: 'DGC_POST_ID', extract: r => String(r.postId) },
  { description: 'Country', extract: r => r.country },
  { description: 'RELEASETYPE', extract: r => r.releaseType },
]

// sources/musicbrainz.ts
tagMappings: [
  { description: 'MusicBrainz Album Id', extract: r => r.extraTags?.['MusicBrainz Album Id'] },
  { description: 'CATALOGNUMBER', extract: r => r.extraTags?.['CATALOGNUMBER'] },
  // ... все动态 из extraTags
]
```

**applyTags** собирает `fields` из tagMappings всех активных источников + editedSiteValues.

## Порядок реализации

1. Шаг 1: Unified SearchResult (types.ts + серверные файлы)
2. Шаг 6: writeUserDefinedText data-driven (tagWriter.ts)
3. Шаг 2: SearchSource интерфейс + source обёртки
4. Шаг 3: Реестр + автогенерация routes
5. Шаг 4: Generic state в useAppContext
6. Шаг 5: SourcePanel + удаление старых компонентов

Каждый шаг — отдельный коммит, проверка пользователем.
