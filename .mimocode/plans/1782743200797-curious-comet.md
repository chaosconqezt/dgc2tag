# Fix: %% матча — разный font-weight в TAGS и TRACKS

## Проблема
`.tc-percent` (TAGS панель) имеет `font-weight: var(--fw-bold)` = 700.
`.mr-sim` (TRACKS панель) наследует 600 от body.
Разница видна: один жирный, другой обычный.

Дополнительно: `.tc-percent` определён дважды в CSS (строки 1361 и 1824), второй дубль мёртвый.

## Исправление

1. **`index.css:1824-1828`** — удалить дублирующий `.tc-percent` (мёртвый, перекрыт первым)
2. **`index.css:2019-2024`** — добавить `font-weight: var(--fw-bold)` в `.mr-sim`

Оба компонента будут использовать 700 (monospace + bold для процентов — хорошо читается).

- `client/src/index.css` — строки 1824-1828 (удалить), строка ~2019 (добавить font-weight)
