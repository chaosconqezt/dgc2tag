# Footer: Music Root + Output Folder path display with folder picker

## Overview
Replace the "Build 012 | Fix TPE1..." footer with a config bar showing music root and output folder paths. Each path has a "Change" button that opens a custom folder picker modal with a browsable directory tree.

## New component: `client/src/components/FolderPicker.tsx`
- Modal with a lazy-loading directory tree
- Uses existing `GET /api/library/children?dirPath=...` endpoint for loading children
- Shows current path at top, directory tree below, "Select" button at bottom
- Supports expanding/collapsing directories
- Props: `initialPath: string`, `onSelect: (path: string) => void`, `onClose: () => void`
- Navigates from root (configMusicRoot) downward, or from any absolute path

## New backend endpoint: `GET /api/directory/roots`
- Returns root drives/paths available on the system (e.g. `C:\`, `D:\`)
- Used by FolderPicker when browsing from absolute path mode (output folder)
- Implementation: `os.platform() === 'win32'` → read drive letters via `wmic logicaldisk get name` or `fs.readdir('/');`
- For music root picker: starts from current musicRoot parent
- For output picker: starts from filesystem root or current output parent

## Modify: `client/src/components/Footer.tsx`
Replace build info with:
```
[Music: C:\music\library ▾] [Output: dgc (subfolder) ▾]
```
- Each section shows the current path/value
- Clicking opens FolderPicker (for music root) or output picker
- Compact horizontal layout, fits in existing footer bar height

## Modify: `client/src/components/SettingsModal.tsx`

- Keep: tag defaults, search sources, cleanup patterns, cache
