# Plan: CSS Rewrite — Unified Modern Panels

## Current State

`index.css` is 2238 lines with duplicated classes, hardcoded px values, redundant `font-family` on every rule, and inconsistent spacing. The user wants a clean, modern, unified design.

## Key Problems

1. **Duplicated classes**: `.tag-dash` and `.tag-value` defined twice (lines 1782+2237, 1786+2238)
2. **`font-family: var(--font)`** on ~80 classes — redundant (body sets it)
3. **Hardcoded px** scattered: `10px`, `14px`, `16px`, `20px`, `32px`, `48px` — should use `--space-*` vars
4. **`border-radius: 0`** on every component — all radius vars already 0, declarations redundant
5. **Inconsistent font sizes**: `12px` instead of `var(--fs-s)`, `14px` instead of `var(--fs)`
6. **Near-identical button classes**: `.btn-ok` / `.btn-ok-red` — should be `.btn-primary`
7. **Overly specific section comments** (`═══════`, `── Section ──`) — clean up to minimal headers
8. **Many unused utility classes** (`.h-screen`, `.absolute`, `.relative`, `.fixed`, `.inset-0`, `.z-1000`, `.z-9999`, `.opacity-30`, `.opacity-60`, `.whitespace-nowrap`, `.word-break`, `.line-through`, `.transition-transform`, `.transition-border`, `.transition-fast`, `.transition-normal`, `.transition-colors`, `.transition-bg`, `.pl-2`, `.pr-2`, `.pt-0`, `.pb-0`, `.m-0`, `.mt-1`, `.mt-2`, `.ml-1`, `.mb-1`, `.text-purple`, `.text-center`, `.text-right`, `.text-upper`, `.fw-400`, `.fw-500`, `.box-border`, `.bg-panel`, `.bg-card`, `.bg-input-alt`, `.bg-input-alt2`, `.bg-transparent`, `.bg-none`, `.rounded-sm`, `.rounded`, `.rounded-lg`, `.rounded-xl`, `.border-none`, `.border-transparent`, `.border-t-light`, `.items-start`, `.fs-48`, `.ls-tight`, `.ls-wide`, `.lh-normal`)

## Approach: Single Rewrite of `index.css`

Replace the entire file with a clean, organized CSS file (~800-1000 lines). No component files need changes — all class names stay the same.

### Structure

```
1. Design Tokens (:root)
2. Reset (body, box-sizing, scrollbar, focus)
3. Utilities (only what's actually used)
4. Layout (dashboard, sidebar, main-content, diff-panel)
5. Tables (unified row/cell/input/percent system for tag-row + track-row)
6. Components (buttons, modals, panels, search, footer, etc.)
7. Domain-specific (library, tree, settings, progress, etc.)
```

### Design Principles

- **font-size**: All body text = `var(--fs)` (14px). Small labels = `var(--fs-s)` (13px). Extra small = `var(--fs-xs)` (10px). No hardcoded px.
- **No border-radius** (all vars = 0)
- **No borders** on containers (only `border-bottom` for row separators)
- **Minimal padding**: rows = `1px 0`, cells = `2px 4px`, panels = `10px`
- **Unified table system**: tag-row and track-row share the same cell/input/hover/edited/mismatch base
- **line-height**: 1.2 everywhere (already set on body)
- **Remove all `font-family`** from individual rules (inherited from body)

### What Stays in Components

Only dynamic inline styles remain:
- `width: sidebarWidth`, `height: treeHeightPx` (App.tsx)
- `accentColor: COLORS.red/green/etc` (checkbox accents)
- `left: x, top: y` (context menu position)
- `width: ${pct}%` (progress bar)
- `color: success ? green : red` (conditional)
- `opacity: readonly ? 0.3 : 1` (conditional)
- `--card-size: ${cardSize}px` (CSS var)

### What Changes in index.css

1. **Remove all `font-family: var(--font)`** from individual rules
2. **Remove all `border-radius: 0`** from individual rules
3. **Remove all `border-radius: var(--radius-*)`** (vars are already 0)
4. **Replace hardcoded `10px`/`14px`/`16px`/`20px`/`32px`** with `--space-*` vars
5. **Remove unused utility classes**
6. **Merge duplicate classes**
7. **Consolidate `.btn-ok`/`.btn-ok-red` into `.btn-primary`**
8. **Clean section headers** to minimal comments
9. **Unify tag-row and track-row** shared base classes
