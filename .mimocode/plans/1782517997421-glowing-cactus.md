# Fix: Infinite loop in `getBandDiscography` pagination

## Problem

`getBandDiscography(bandId)` in `server/src/scraper.ts:572-613` gets stuck in an infinite loop for band 5807.

**Root cause:** Line 589 blindly trusts the DGC API's `discography.offset` field:
```ts
offset = data.band.discography?.offset ?? offset + 1;
```

The DGC API for band 5807 returns `hasMore: true` with `offset: 1` on every request — the offset never advances. The code fetches `offset=1` repeatedly with 1.5s delays forever, never terminating.

**Log evidence:** Requests alternate between `offset=0` (first page) then stuck on `offset=1` indefinitely.

## Fix

**File:** `server/src/scraper.ts` — `getBandDiscography()` (lines 572-613)

Three safeguards in the pagination loop:

1. **Detect non-advancing offset:** If the API returns the same offset as the previous request, advance manually by the number of posts returned (or break if 0 posts).
2. **Detect empty page:** If `discography.posts` is empty, break regardless of `hasMore`.
3. **Hard iteration limit:** Add a `MAX_PAGES` constant (e.g., 100) as a safety net.

### Changed code (lines 576-591):

```ts
const allPosts: DgcPost[] = [];
let offset = 0;
let hasMore = true;
let bandName = '';
const MAX_PAGES = 100;

while (hasMore) {
    const data = await apiFetch<{ ... }>(`/api/bands/${bandId}?offset=${offset}`);
    if (!data.band) return null;
    bandName = data.band.name;
    const posts = data.band.discography?.posts || [];
    allPosts.push(...posts);
    hasMore = posts.length > 0 && (data.band.discography?.hasMore ?? false);
    const nextOffset = data.band.discography?.offset;
    if (nextOffset != null && nextOffset > offset) {
        offset = nextOffset;
    } else {
        offset += posts.length || 1;
    }
    if (hasMore) await new Promise(r => setTimeout(r, 1500));
    if (allPosts.length > 0 && --MAX_PAGES <= 0) break;  // safety net
}
```

Key changes:
- `hasMore` now also requires `posts.length > 0` — empty page = done
- Offset advancement: if API's `nextOffset` is not greater than current, increment by `posts.length` (or +1 as fallback)
- Hard safety limit of 100 iterations
